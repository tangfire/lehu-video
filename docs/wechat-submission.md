# 微信小程序提审与社区规则

这份文档列出校园 e站提审前需要准备的非代码内容，以及小程序后台必须配置的域名和隐私项。

## 提审前必须准备

至少准备这些真实内容：

| 内容 | 说明 |
| --- | --- |
| 隐私保护指引 | 覆盖头像、昵称、相册/相机、网络请求、反馈、内容发布 |
| 用户协议 | 说明账号、内容、社区秩序、责任边界 |
| 社区规范 | 发帖、评论、举报、违规处理规则 |
| 联系方式 | 反馈、投诉、侵权、内容处理联系入口 |
| 测试账号 | 给微信审核人员可登录和体验 |
| 内容安全说明 | 说明人工审核、AI 初审、举报反馈、下架机制 |

这些不是代码问题，但会直接影响提审。

## 隐私保护指引要覆盖什么

建议按功能写清楚：

| 能力 | 涉及信息 |
| --- | --- |
| 微信登录 | openid、unionid、头像、昵称 |
| 用户资料 | 昵称、头像、学校身份资料 |
| 发帖评论 | 文本、图片、互动记录 |
| 图片上传 | 相册/相机选择的图片 |
| 反馈举报 | 反馈内容、举报理由、联系方式或截图 |
| 通知 | 站内消息、互动通知 |
| 安全风控 | IP、User-Agent、请求路径、request_id |
| e仔/RAG | 用户提问、帖子上下文、命中知识库片段 |

如果小程序实际没有申请某项权限，就不要在指引里过度声明；如果代码里用了，就必须在隐私里说明。

## 微信后台域名

小程序后台需要配置：

| 类型 | 域名 |
| --- | --- |
| request 合法域名 | API 域名，例如 `https://api.example.com` |
| uploadFile 合法域名 | COS 上传域名，例如 `https://campus-1250000000.cos.ap-guangzhou.myqcloud.com` |
| downloadFile 合法域名 | CDN 下载域名，例如 `https://cdn.example.com` |

如果运营后台生成朋友圈素材里的小程序码，生产还需要：

```bash
WECHAT_APP_ID=...
WECHAT_APP_SECRET=...
WECHAT_MINIPROGRAM_QR_ENV_VERSION=release
```

## 社区规范建议包含

社区规则至少写清：

- 禁止违法违规、色情暴力、广告诈骗、恶意引战。
- 禁止泄露他人隐私，例如手机号、住址、身份证、聊天记录。
- 禁止代考代课、买卖危险物品、造谣传谣。
- 校园政策以学校官方渠道为准，平台内容仅供交流。
- 用户可以举报帖子和评论。
- 运营方可以审核、下架、删除违规内容，严重时限制账号。
- e仔回答只作为辅助，涉及正式事项以学校官方通知为准。

## 审核机制说明

当前系统支持：

| 机制 | 说明 |
| --- | --- |
| 发帖审核开关 | 不审核、人工审核、AI 初审 |
| 举报处理 | 用户可以举报帖子/评论，后台处理 |
| 用户反馈 | 用户可以提交反馈，后台跟进 |
| 内容下架 | 运营可下架帖子、删除评论 |
| IP 封禁 | 异常请求可封禁 IP |

提审材料里可以说明平台有内容治理能力，但不要承诺“所有内容实时自动审核”，因为实际策略会按后台开关变化。

## 功能验收清单

提审前至少手动走一遍：

1. 微信真实登录成功。
2. 修改头像/昵称成功。
3. 发文字帖成功。
4. 发图片帖成功，图片来自 CDN。
5. 视频入口不可见，视频帖被后端拒绝。
6. 评论、点赞、收藏成功。
7. 举报帖子和用户反馈成功。
8. 通知列表和未读数正常。
9. 扫朋友圈素材小程序码能打开帖子详情。
10. 退出重进后登录态正常。

## 审核测试账号

建议给微信审核准备一个普通用户账号和必要说明：

```text
测试账号：
测试方式：
可体验功能：浏览帖子、发帖、评论、反馈、举报、通知
不开放功能：视频发布、私聊、短视频
```

不要给微信审核后台管理员账号，除非审核要求运营端能力。

## 需要避免的提审风险

- 页面里出现“短视频”“聊天 IM”等旧项目残留文案。
- 隐私指引没覆盖头像、相册/相机、反馈、发帖。
- 图片上传域名没配置，导致审核人员发图失败。
- 内容治理说明缺失，社区类小程序容易被打回。
- e仔回答暗示“学校官方最终解释”，需要强调不代表学校官方。
- 朋友圈素材功能被描述成“自动发朋友圈”，实际不支持。
